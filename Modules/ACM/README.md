# AWS ACM Terraform Module

This module provisions an AWS Certificate Manager (ACM) certificate and performs automatic DNS validation using Amazon Route53.

The module:

- Creates an ACM certificate
- Supports Subject Alternative Names (SANs)
- Creates Route53 validation records automatically
- Performs ACM certificate validation
- Exposes the validated certificate ARN for use with ALB, CloudFront, and other AWS services

---

## Features

- Automated DNS validation
- Route53 integration
- Support for wildcard certificates
- Support for multiple SANs
- Create-before-destroy lifecycle to prevent downtime
- Standardized tagging

---

| Name | Version |
|------|---------|

## Usage

### Single Domain Certificate

```hcl
module "acm" {
  source = "../Modules/ACM"

  project_name = "saas"
  environment  = "production"

  domain_name = "example.com"
  zone_id     = module.r53.hosted_zone_id
}
```

---

### Wildcard Certificate

```hcl
module "acm" {
  source = "../Modules/ACM"

  project_name = "saas"
  environment  = "production"

  domain_name = "*.example.com"
  zone_id     = module.r53.hosted_zone_id
}
```

---

### Certificate with Subject Alternative Names

```hcl
module "acm" {
  source = "../Modules/ACM"

  project_name = "saas"
  environment  = "production"

  domain_name = "example.com"

  subject_alternative_names = [
    "*.example.com",
    "api.example.com",
    "admin.example.com"
  ]

  zone_id = module.r53.hosted_zone_id
}
```

---

## Inputs

| Name | Type | Description | Required |
|--------|--------|-------------|----------|
| project_name | `string` | Project name. | Yes |
| environment | `string` | Environment name. | Yes |
| domain_name | `string` | Primary domain name for the certificate. | Yes |
| zone_id | `string` | Route53 Hosted Zone ID used for DNS validation. | Yes |
| subject_alternative_names | `list(string)` | Additional domain names (SANs) for the certificate. | No |

---

## Example Input

```hcl
domain_name = "example.com"

subject_alternative_names = [
  "*.example.com"
]
```

---

## Outputs

| Name | Description |
|--------|-------------|
| certificate_arn | ARN of the validated ACM certificate |
| domain_validation_options | Validation records generated by ACM |

### Example

```hcl
module "alb" {
  source = "../Modules/ALB"
```

## Validation Process

The module automatically performs the following steps:

```text
ACM Certificate Request
          │
          ▼
Generate Validation Records
          │
          ▼
Create Route53 Records
          │
          ▼
ACM Certificate Validation
          │
          ▼
Certificate Issued
```

## Wildcard Certificate Example

```hcl
domain_name = "example.com"

subject_alternative_names = [
  "*.example.com"
]
```

This certificate can secure:

- app.example.com
- api.example.com
- admin.example.com
- any-subdomain.example.com

---

## Best Practices

- Use DNS validation instead of Email validation.
- Use wildcard certificates when multiple subdomains are required.
- Store Route53 Hosted Zone in a dedicated module.
- Always reference the validated certificate ARN output.
- Avoid manually creating validation records outside Terraform.
- Use `create_before_destroy` to prevent service interruptions during certificate replacement.

## Notes

- DNS validation records are created automatically.
- Route53 Hosted Zone must already exist.
- Certificate issuance may take several minutes after DNS propagation.
- ACM certificates used with an ALB must exist in the same AWS Region as the ALB.
- Wildcard certificates should be added using `subject_alternative_names` when both root and wildcard domains are required.

Example:

```hcl
domain_name = "example.com"

subject_alternative_names = [
  "*.example.com"
]
```
